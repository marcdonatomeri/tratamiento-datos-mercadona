Algoritmo del proyecto.

Introducción.

Este proyecto tiene como objetivo desarrollar un algoritmo en el lenguaje de programación R capaz de realizar un análisis exhaustivo de tickets de compra en formato PDF, específicamente de los obtenidos en establecimientos de Mercadona. A través de este proceso, se busca extraer la información relevante de cada ticket y convertirla en un formato estructurado, como lo es el data frame, que facilita su análisis y manipulación.

La importancia de este proyecto radica en la posibilidad de automatizar el proceso de recopilación de datos de tickets en formato PDF, que a menudo son difíciles de manejar debido a su estructura no estandarizada. El uso de R para esta tarea ofrece una gran flexibilidad y potencia a la hora de trabajar con datos y aplicar técnicas de análisis estadístico y visualización.

El enfoque del proyecto se basa en la creación de un algoritmo eficiente que sea capaz de leer, interpretar y extraer información clave de los tickets, tales como: fecha de la compra, productos adquiridos, cantidades, precios y, especialmente, el impacto de factores como el IVA en el precio final de los productos. Este análisis podría proporcionar valiosas perspectivas sobre cómo ha afectado la evolución del IVA a los precios de productos específicos, lo que permitiría realizar comparativas a lo largo del tiempo.

- ¿Qué tipo de carne (ternera, cerdo, pollo o pavo) ofrece mejor relación calidad-precio según la frecuencia de compra y el precio promedio?

# Análisis de relación calidad-precio por tipo de carne
relacion_calidad_precio <- productos_global %>%
  # Filtrar productos cárnicos principales
  filter(str_detect(toupper(Descripcion), "TERNERA|CERDO|POLLO|PAVO|VACUNO")) %>%
  mutate(
    # Clasificar por tipo de carne
    tipo_carne = case_when(
      str_detect(toupper(Descripcion), "TERNERA|VACUNO") ~ "Ternera",
      str_detect(toupper(Descripcion), "CERDO") ~ "Cerdo",
      str_detect(toupper(Descripcion), "POLLO") ~ "Pollo",
      str_detect(toupper(Descripcion), "PAVO") ~ "Pavo",
      TRUE ~ "Otro"
    ),
    P.Unit = as.numeric(P.Unit)
  ) %>%
  group_by(tipo_carne) %>%
  summarise(
    Precio_Promedio = mean(P.Unit, na.rm = TRUE),
    Frecuencia_Compra = n(),
    Relacion_Calidad_Precio = Frecuencia_Compra / Precio_Promedio
  ) %>%
  arrange(desc(Relacion_Calidad_Precio))

# Visualización

# Calculamos, para cada ticket, el porcentaje de IVA que representa parte del total.
ticket_con_iva <- tickets_global %>%
  mutate(
    # Convertimos el importe total de carácter a número, reemplazando la coma decimal por punto.
    importe_total_num = as.numeric(gsub(",", ".", importe_total)),
    
    # Extraemos todos los porcentajes de IVA (ej. "10%", "21%") y nos quedamos con el valor máximo,
    # en caso de existir varios en un mismo ticket.
    max_iva = sapply(str_extract_all(tipos_iva, "\\d+%"), function(x) {
      # Convertimos cada valor (eliminando el símbolo "%") a número y seleccionamos el máximo.
      max(as.numeric(str_remove(x, "%")), na.rm = TRUE)
    }),
    
    # Calculamos el porcentaje del total que corresponde al IVA.
    # Si un ticket tiene tasa R, entonces, el IVA incluido en el total es R/(100+R)*100.
    porcentaje_iva = (max_iva / (100 + max_iva)) * 100
  )

# Calculamos el porcentaje promedio de IVA para todos los tickets.
porcentaje_iva_promedio <- ticket_con_iva %>%
  summarise(porcentaje_promedio = mean(porcentaje_iva, na.rm = TRUE))

  • ¿Se puede detectar algún cambio en los hábitos de consumo antes y después de periodos vacacionales?
# Añadir fechas a los productos y clasificar cada compra según su periodo (pre-vacacional, vacacional, etc.)
gasto_por_periodo <- productos_global %>%
  left_join(tickets_global %>% select(factura_simplificada, fecha), by = "factura_simplificada") %>%
  mutate(fecha = as.Date(fecha, format = "%d/%m/%Y")) %>%
  mutate(tipo_periodo = case_when(
    fecha >= as.Date("2023-12-05") & fecha < as.Date("2023-12-20") ~ "Pre-vacacional",
    fecha >= as.Date("2023-12-20") & fecha <= as.Date("2024-01-07") ~ "Vacacional",
    fecha > as.Date("2024-01-07") & fecha <= as.Date("2024-01-22") ~ "Post-vacacional",
    TRUE ~ "Normal"
  )) %>%
  
# Calcular el gasto total por ticket y clasificarlo por tipo de periodo
  group_by(factura_simplificada, tipo_periodo) %>%
  summarise(total_ticket = sum(P.Unit, na.rm = TRUE), .groups = "drop") %>%
# Calcular el gasto promedio por tipo de periodo
  group_by(tipo_periodo) %>%
  summarise(
    gasto_promedio = mean(total_ticket, na.rm = TRUE),
    tickets = n()
  ) %>%
  arrange(desc(gasto_promedio))

A partir del análisis realizado, podemos concluir que el algoritmo desarrollado en R es capaz de extraer y estructurar de manera eficaz la información contenida en los tickets de compra en PDF de Mercadona. Esto ha permitido observar patrones relevantes en el comportamiento de los precios y el gasto.

En particular, se ha identificado una variación significativa en los precios de ciertos productos a lo largo del tiempo, lo que podría estar relacionado con ajustes en el IVA y otros factores económicos. Además, se observó que el impacto del IVA no siempre se transfiere de forma uniforme al precio final, sugiriendo estrategias comerciales o subsidios específicos por producto.

También se logró determinar los periodos con mayor gasto promedio por ticket, lo cual podría asociarse a fechas especiales o promociones. Finalmente, la automatización de este proceso representa una herramienta útil para estudios de consumo, facilitando análisis longitudinales que serían inviables de realizar manualmente.
